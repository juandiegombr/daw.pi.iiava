# Contributing Guide

Thank you for considering contributing to this project!

## How to Contribute

### 1. Fork the Repository

Create your own fork of the project.

### 2. Clone Your Fork

```bash
git clone https://github.com/your-username/repository.git
```

### 3. Create a Branch

```bash
git checkout -b feature/my-feature
```

### 4. Make Your Changes

Ensure your code follows the project standards and includes tests when
possible.

### 5. Commit Your Changes

```bash
git commit -m "Add: description of changes"
```

### 6. Push to Your Fork

```bash
git push origin feature/my-feature
```

### 7. Open a Pull Request

Submit your pull request through GitHub.

## Code Style

- Follow existing conventions.
- Write clear, readable code.
- Add comments where necessary.

## Reporting Bugs

Use GitHub Issues and provide: - Clear description - Steps to
reproduce - Expected vs actual behavior

## Feature Requests

Open an issue describing: - Motivation - Use case - Possible
implementation

## Pull Request Guidelines

- One feature/fix per PR
- Reference related issues
- Ensure CI passes

## Code of Conduct

Please be respectful and inclusive.
